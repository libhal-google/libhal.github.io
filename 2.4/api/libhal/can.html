inline status bus_on()#

Transition the CAN device from ‚Äúbus-off‚Äù to ‚Äúbus-on‚Äù.

Can devices have two counters to determine system health. These two counters are the ‚Äútransmit error counter‚Äù and the ‚Äúreceive error counter‚Äù. Transmission errors can occur when the device attempts to communicate on the bus and either does not get an acknowledge or sees an unexpected or erroneous signal on the bus during its own transmission. When transmission errors reach 255 counts, the device will go into the ‚Äúbus-off‚Äù state.

In the ‚Äúbus-off‚Äù state, the CAN peripheral can no longer communicate on the bus. Any calls to send() will throw the error std::errc::network_down. If this occurs, this function must be called to re-enable bus communication.

Warning

Calling this function when the device is already in ‚Äúbus-on‚Äù will have no effect. This function is not necessary to call after creating the CAN driver as the driver should already be ‚Äúbus-on‚Äù on creation.

Returns:

status - success or failure. In the case this function fails repeatedly, it is advised to simply not use the bus anymore as something is critical wrong and may not be recoverable.

std::uint8_t phase_segment1 = 3#

Length of Phase Segment 1 (1qt ‚Ä¶ 8qt)

Determines the bit rate, phase segment 1 acts as a buffer that can be lengthened to resynchronize with the bit stream via the synchronization_jump_width. Includes propagation delay

std::uint8_t phase_segment2 = 3#

Length of Phase Segment 2 (1qt ‚Ä¶ 8qt)

Determines the bit rate and is like phase segment 1 and occurs after the sampling point. Phase segment 2 can be shortened to resynchronize with the bit stream via the synchronization_jump_width.

std::uint8_t synchronization_jump_width = 1#

Synchronization jump width (1qt ‚Ä¶ 4qt)

This is the maximum time by which the bit sampling period may be lengthened or shortened during each cycle to adjust for oscillator mismatch between nodes.

This value must be smaller than phase_segment1 and phase_segment2

constexpr std::optional<std::uint32_t> is_valid(const can::settings &p_settings, hertz p_operating_frequency)#

Validate configuration settings against an operating frequency.

The settings and frequency must follow the following rules:

  1. propagation_delay, phase_segment1, phase_segment2 and synchronization_jump_width must be nonzero.

  2. synchronization_jump_width must be the lesser between phase_segment1 and phase_segment2.

  3. The total bit width must be equal to or greater than 8 Tq/bit; the sum of sync_segment, propagation_delay, phase_segment1 and phase_segment2.

  4. The CAN device‚Äôs operating frequency must be at least 8 times the baud rate to give the minimum.

  5. The ratio between the CAN device‚Äôs operating frequency and the bit width must be close enough to an integer to produce a usable baud rate prescaler.

Parameters:

  • p_settings ‚Äì - settings object to check

  • p_operating_frequency ‚Äì - CAN device operating frequency

Returns:

std::optional<std::uint32_t> - baud rate prescaler
